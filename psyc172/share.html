技术细节

找到的 Bug 和修复过程

这个项目最有意思的部分——一个隐藏很深的 HTML 解析 Bug。

🐛

主 Bug：所有按钮完全不可点击

所有导航按钮、闪卡、测验全部点击无响应。初步怀疑是 JS 函数定义问题或 CSS 覆盖，但经过诊断发现根因更隐蔽：

真正原因： JS 字符串里存在 p<0.05，HTML 解析器遇到 <0 时将其识别为标签开头，导致整个 <script> 块在中途被截断——后续所有 JS 函数定义全部失效。

// ❌ 问题代码（HTML 解析器会把 <0 当标签）
exp: "当 p<0.05 时结果显著..."

// ✅ 修复方案（使用全角小于号 U+FF1C，功能完全一致）
exp: "当 p＜0.05 时结果显著..."

// 验证命令
node -e "new Function(scriptCode)" → All scripts OK ✓
⚠️

次要 Bug：JS 字符串内未转义双引号

在写闪卡和测验的中文内容时，多处出现在双引号字符串内部包含双引号的情况，比如 "游群"是什么、"美即好"效应、称之为"读心盲" 等 7 处，导致 JS 语法错误。

// ❌ 问题
{q: "[W1-Q12] "游群（Bands）"是什么？"}

// ✅ 修复：换成书名号或去掉引号
{q: "[W1-Q12] 游群（Bands）是什么？"}

{a: "「美即好」（Dion等1972）..."}
🔍

路径 Bug：目录名含未闭合括号

PDF 文件路径 Spring 26 (Junior) 实际上磁盘上的目录名是 Spring 26 (Junior（没有右括号）——用 find 命令确认后才正确读取到 PDF。
